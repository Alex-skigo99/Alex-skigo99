Full Stack Application · B2B Platform

Personality Collection

A full-stack web platform that assesses YouTube creators' personality types through a structured Big 5 personality survey and delivers AI-generated, personalised growth recommendations. Designed as a B2B tool for agencies or networks that manage YouTuber talent.

Business Logic
  • Invitation Flow — YouTuber receives a branded email with a unique survey link containing their email and channel URL as query parameters
  • Survey Instance Creation — When the email logo loads (image tracking pixel), the system initialises a new survey record on the backend
  • Personality Assessment — Survey based on the Big 5 Personality Model with individual answer saving for resume-from-where-you-left-off support
  • Engagement Mechanics — Motivational "meme breaks" every 5 questions with progress-based encouragement
  • Channel Verification — Post-survey modal for YouTube channel ownership confirmation
  • AI-Powered Results — LLM generates a multi-section report: creator type, superpower, weakness, growth tip, personality summary, and growth strategy
  • Viral Sharing — Users can share the survey with other creators via email from the result page
Architecture

Key Design Decisions

  • Individual Answer Saving — Each response is PATCHed independently. No data loss if the user closes the browser, plus full resume support
  • React Query over Redux for Server State — Survey data managed by React Query; Redux is intentionally minimal (loader only)
  • Interceptor Pattern — A single InterceptorsComponent wraps the app tree, setting up Axios interceptors once with a global Spin overlay
  • Error Class Hierarchy — Base HRCustomError with getLoggerObject() and convertToResponse() methods. Subclasses carry domain-specific status codes
  • Cached AI Results — Personality report generated once and stored in survey document. Subsequent requests return cached result
  • Email Tracking Pixel — Invitation email logo URL contains creator info. Image load triggers survey creation — no user interaction required
  • In-Memory MongoDB for Tests — mongodb-memory-server with disposable replica set per test run
