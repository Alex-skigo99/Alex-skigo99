Full Stack Application

Nail Salon Management

A full-stack web application designed to automate operations for beauty businesses — nail salons, hairdressers, beauty studios, and similar establishments. The system features a client-facing website for booking and account management, and an admin panel for end-to-end business management.

Business Logic

Client-Facing Portal

  • Public landing page with salon info, photo gallery, price list, and contact form
  • Appointment booking — clients select a service provider, date, and available time slot
  • Booking available to both guests (no account required) and authenticated users
  • Client dashboard for tracking bookings, viewing appointment history, and managing account details
  • Multi-language support — English, Russian, and Hebrew (RTL)
  • Authentication — email/password registration and Google OAuth via next-auth v5

Admin Panel

  • Staff management — create, edit, deactivate service providers with per-weekday schedules
  • Services & pricing — manage services with categories (manicure, pedicure, other), prices, and durations
  • Products management — create and manage additional client offerings
  • Client management — searchable list with full profile view and appointment history
  • Appointment calendar — weekly and monthly views for visual planning
  • Appointments table — filterable, sortable list with full CRUD operations
  • Email notifications — individually configurable per master via AWS SES
  • Automated appointment reminders
  • Configurable business-level settings
Architecture

Backend Architecture

Clean layered architecture following the pattern: Request → CORS/cookieParser → Route → authenticate (JWT) → requireRole → Controller (Zod validation) → Service (business logic + Knex queries) → Response

  • Routes — thin Express routers, HTTP verb + path mapping only
  • Controllers — parse and validate input (Zod), call the service, return responses
  • Services — all business logic, DB queries (Knex), and external API calls
  • Middleware — authenticate (JWT), requireRole (RBAC), centralized error handler

Frontend Architecture

App Router (Next.js) → Server Components → Client Components → Custom Hooks (TanStack Query) → API Client (Axios) → Backend REST API

  • Server state managed by TanStack Query with centralised query keys
  • Custom hooks encapsulate all data fetching and mutations
  • Forms use React Hook Form + Zod resolver for type-safe validation
  • Separate layouts for admin (collapsible shadcn Sidebar) and client sections
  • Admin routes English-only; client routes use next-intl with three locales

Dual Lambda Deployment

  • nail-saloon-backend — handles all HTTP requests via API Gateway + serverless-http
  • MigrationLambda — dedicated function for Knex database migrations
